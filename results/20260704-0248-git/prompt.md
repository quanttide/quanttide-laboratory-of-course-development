请为 "git" 设计一份课程蓝图。

## 背景

课程蓝图是课程核心思路的概念框架。它回答的不是"教什么"，而是"为什么这样教"。

我们遵循的教学哲学是**追本溯源**——不是从结论出发，而是回到知识和工具被创造时的原始场景，理解它们解决了什么问题、为什么被设计成今天的样子。

## 要求

请按以下结构输出：

1. **起源**：git 是怎么诞生/发明出来的？当时面对的核心问题是什么？
2. **核心对比**：用一个对比框架帮助理解（例如：分布式 vs 中心化）
3. **关键概念**：列出最关键的概念，并按逻辑顺序排列（而非平铺罗列）
4. **教学流程**：从原理推导到具体操作的教学步骤
5. **常见误区**：初学者最容易困惑的地方，以及如何化解

## 风格

- 先讲"为什么"，再讲"是什么"，最后讲"怎么用"
- 把容易混淆的概念拆开单独讲
- 用真实场景或类比来辅助理解
