# 实验：LLM 生成课程蓝图

## 实验目的

测试 LLM 能否基于"追本溯源"的教学哲学，生成符合量潮风格的课程蓝图。由人类讲师对结果打分。

## 流程

1. **选题**：选择一个知识点/工具（如 Git、Docker、Python 等）
2. **Prompt**：向 LLM 描述课程蓝图的定义和核心方法，要求它生成该主题的课程蓝图
3. **生成**：记录 LLM 的原始输出
4. **评分**：人类讲师按以下维度打分
5. **结论**：判断 LLM 是否掌握了"溯源"思路

## 评分维度（每项 1-5 分）

| 维度 | 说明 |
|------|------|
| 溯源深度 | 是否追溯到问题/工具的起源和设计场景 |
| 概念对比 | 是否使用了对比框架（如分布式 vs 中心化） |
| 逻辑链条 | 知识点之间是否有因果逻辑而非平铺罗列 |
| 教学可操作性 | 生成的蓝图是否能直接指导备课 |
| 风格契合度 | 是否体现"道先于术"的量潮风格 |
